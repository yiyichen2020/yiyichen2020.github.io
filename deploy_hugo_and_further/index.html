Deploy Hugo and Further

The following operations is based on the post, “Hugo With LoveIt Theme Configuration”, which requires you have already installed and config both Hugo and LoveIt theme.

1. Deploy Hugo on Github

  1. Assume you have already got a Github account.

  2. Create a Github Repository, name this repository as yourgithubname.github.io.

  3. Open your terminal, and cd to your blog folder.

  4. Run following command lines:

1
2
3
4
5
6
7
8

hugo --theme=loveit --baseUrl="https://yourgithubname.github.io/"
 # To build static pages, which will generate public folder
cd public
git init
git remote add origin https://github.com/yourgithubname/yourgithubname.github.io.git
git add .
git commit -m "first commit"
git push
  1. After a few seconds, your blog is online!

2.Advance

1. Add Comment Box to Your Blog

1-1. Install utterances on your repo.

The easiest way for you to add comment box is adding utterances.es comment box. Here is the brief intruduction of utterances.es: https://utteranc.es/

1-2. Open quickstart/config.toml file, and set utterances.es related setting like following.

[params.page.comment.utterances]
    enable = true
    # owner/repo
    repo = "yourgithubname/yourgithubname.github.io"
    issueTerm = "pathname"
    label = ""
    lightTheme = "github-light"
    darkTheme = "github-dark"

1-3. Set enable = false under [params.page.comment.valine]

1-4. Rebuild your blog and redeploy your public folder.

1-5. Enjoy commentingüçª

2. Add Advanced Search (Algolia) Ability to Your Blog

2-1. Create algolia account: https://www.algolia.com/

2-2. Create an application under your algolia account; [Choose Experienced User, continue all the way down]

2-3. Configure your application; [Click Indice in the left sidebar, click Create Index in the right sidebar just pop out, choose a name you like as Index name]

2-4. Add your algolia application related API Keys into your config.toml file. You can find API Key on the sidebar in algolia website.

3. A Proper Way to Add Image in Your Post

  1. Create a folder after your post’s name under static folder; For example your post’s file name is proper.md, so your image1 in that post should under quickstart/static/proper/image1.png
  2. Add ![image1](image1.png) into proper.md
  3. Rebuild and redeploy your blog, the image1 will show!

package main

import (
	"encoding/json"
	"fmt"
	"io/ioutil"
	"os"

	"github.com/algolia/algoliasearch-client-go/v3/algolia/opt"
	"github.com/algolia/algoliasearch-client-go/v3/algolia/search"
)

type Post struct {
	Categories []string `json:"categories"`
	Content    string   `json:"content"`
	Date       string   `json:"date"`
	ObjectID   string   `json:"objectID"`
	Tags       []string `json:"tags"`
	Title      string   `json:"title"`
	Uri        string   `json:"uri"`
}

func main() {
	var posts []Post
	client := search.NewClient("Your Own Application ID", "Your Own Admin API Key
")
	index := client.InitIndex("indexname")
	file, _ := os.Open("./public/index.json")
	data, _ := ioutil.ReadAll(file)
	_ = json.Unmarshal(data, &posts)
	fmt.Println(posts)
	index.SaveObjects(posts, opt.AutoGenerateObjectIDIfNotExist(true))
}

Reference link: https://www.algolia.com/doc/guides/sending-and-managing-data/send-and-update-your-data/how-to/importing-with-the-api/?client=go
